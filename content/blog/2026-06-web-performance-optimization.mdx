## Core Web Vitals：Google 的性能三件套

Google 用三个核心指标衡量网页体验，直接影响搜索排名：

| 指标 | 全称 | 含义 | Good | Poor |
| --- | --- | --- | --- | --- |
| LCP | Largest Contentful Paint | 最大内容绘制时间 | < 2.5s | > 4.0s |
| INP | Interaction to Next Paint | 交互到下一次绘制 | < 200ms | > 500ms |
| CLS | Cumulative Layout Shift | 累积布局偏移 | < 0.1 | > 0.25 |

### LCP（最大内容绘制）

通常是首屏中最大的图片或文字块。优化 LCP 是首屏体验的核心。

### INP（交互到下次绘制）

2024 年 3 月正式替代 FID，衡量页面对用户交互的响应速度。点击、键盘输入、点按都要在 200ms 内给出视觉反馈。

### CLS（累积布局偏移）

页面加载过程中元素的意外移动。最常见的元凶是图片没有尺寸、字体加载导致文字跳动、动态插入内容。

## LCP 优化：让首屏内容更快出现

### 1. 识别 LCP 元素

```javascript
// 用 PerformanceObserver 监听 LCP
new PerformanceObserver((list) => {
  const entries = list.getEntries();
  const lastEntry = entries[entries.length - 1];
  console.log('LCP element:', lastEntry.element);
  console.log('LCP time:', lastEntry.startTime);
  console.log('LCP type:', lastEntry.renderTime ? 'render' : 'load');
}).observe({ type: 'largest-contentful-paint', buffered: true });
```

### 2. 关键渲染路径优化

LCP 元素的加载链路通常是：HTML → CSS → 图片/字体 → 渲染。优化每一环：

```html
<head>
  <!-- 预连接到图片 CDN -->
  <link rel="preconnect" href="https://cdn.example.com">
  <link rel="dns-prefetch" href="https://cdn.example.com">

  <!-- 预加载 LCP 图片 -->
  <link rel="preload" as="image" href="/images/hero.jpg" fetchpriority="high">

  <!-- 关键 CSS 内联（减少渲染阻塞） -->
  <style>
    /* 首屏关键样式内联 */
    .hero { min-height: 50vh; background: var(--bg); }
  </style>

  <!-- 非关键 CSS 异步加载 -->
  <link rel="preload" as="style" href="/css/non-critical.css" onload="this.rel='stylesheet'">
</head>
```

### 3. 图片优化

```html
<!-- 响应式图片 + 懒加载 -->
<img

  alt="Hero"
```

### 4. 字体优化

字体加载会阻塞文本渲染（FOIT — Flash of Invisible Text），导致 LCP 延迟。

```css
/* 使用 font-display: swap 避免 FOIT */
@font-face {
  font-family: 'Noto Sans SC';
  src: url('/fonts/noto-sans-sc.woff2') format('woff2');
  font-display: swap;  /* 先用系统字体，字体加载后替换 */
  font-weight: 400;
  unicode-range: U+0000-00FF, U+4E00-9FFF;  /* 只加载需要的字符范围 */
}
```

**Next.js next/font 自动优化：**

```tsx
import { Noto_Sans_SC, JetBrains_Mono } from 'next/font/google';

const notoSansSC = Noto_Sans_SC({
  subsets: ['latin'],
  weight: ['400', '700'],
  display: 'swap',
  variable: '--font-noto-sans-sc',
});

// next/font 会：
// 1. 自动下载字体文件并自托管
// 2. 注入 CSS 变量
// 3. 使用 font-display: swap
// 4. 预加载字体文件
```

### 5. 减少 JavaScript 阻塞

```html
<!-- 非关键 JS 延迟加载 -->
<script src="/js/analytics.js" defer></script>
<script src="/js/chat-widget.js" async></script>

<!-- 关键 JS 内联 -->
<script>
  // 主题初始化（避免 FOUC）
  (function() {
    const theme = localStorage.getItem('theme') ?? 'system';
    if (theme === 'dark' || (theme === 'system' && matchMedia('(prefers-color-scheme: dark)').matches)) {
      document.documentElement.classList.add('dark');
    }
  })();
</script>
```

## INP 优化：让交互更流畅

INP 衡量的是从用户交互到屏幕更新的完整延迟。长任务是 INP 的主要敌人。

### 1. 分解长任务

```javascript
// 坏：一个 200ms 的长任务阻塞主线程
function processAll(items) {
  items.forEach(item => {
    heavyComputation(item);
  });
}

// 好：用 scheduler.yield() 分解任务
async function processAll(items) {
  for (const item of items) {
    heavyComputation(item);
    // 让出主线程
    if ('scheduler' in window && 'yield' in scheduler) {
      await scheduler.yield();
    } else {
      await new Promise(r => setTimeout(r, 0));
    }
  }
}
```

### 2. 使用 requestIdleCallback

```javascript
// 非紧急任务放到空闲时间执行
function prefetchLinks() {
  requestIdleCallback(() => {
    document.querySelectorAll('a[href]').forEach(link => {
      // 预取链接
    });
  }, { timeout: 2000 });
}
```

### 3. 防抖与节流

```typescript
// 防抖：搜索输入
function debounce<T extends (...args: any[]) => void>(fn: T, delay: number) {
  let timer: ReturnType<typeof setTimeout>;
  return (...args: Parameters<T>) => {
    clearTimeout(timer);
    timer = setTimeout(() => fn(...args), delay);
  };
}

// 节流：滚动事件
function throttle<T extends (...args: any[]) => void>(fn: T, limit: number) {
  let inThrottle = false;
  return (...args: Parameters<T>) => {
    if (!inThrottle) {
      fn(...args);
      inThrottle = true;
      setTimeout(() => (inThrottle = false), limit);
    }
  };
}

// 使用
const handleSearch = debounce((query: string) => fetchResults(query), 300);
const handleScroll = throttle(() => updateProgress(), 16);  // ~60fps
```

### 4. React 中的优化

```tsx
// 使用 useTransition 标记非紧急更新
import { useTransition, useDeferredValue } from 'react';
```

## CLS 优化：避免布局跳动

### 1. 图片和嵌入内容预留空间

```css
/* 图片必须设置宽高或 aspect-ratio */
img {
  width: 100%;
  aspect-ratio: 16 / 9;  /* 预留空间 */
}

/* 视频容器 */
.video-container {
  position: relative;
  aspect-ratio: 16 / 9;
}
.video-container iframe {
  position: absolute;
  inset: 0;
}
```

### 2. 字体加载策略

```css
/* 使用 font-display: swap + size-adjust 减少字体替换跳动 */
@font-face {
  font-family: 'Noto Sans SC';
  src: url('/fonts/noto.woff2') format('woff2');
  font-display: swap;
  size-adjust: 100%;  /* 调整 fallback 字体大小匹配自定义字体 */
  ascent-override: 90%;
  descent-override: 20%;
  line-gap-override: 0%;
}
```

### 3. 动画使用 transform

```css
/* 坏：修改 height 导致重排 */
.bad-animation {
  transition: height 0.3s;
}
.bad-animation:hover {
  height: 200px;
}

/* 好：使用 transform 不触发重排 */
.good-animation {
  transition: transform 0.3s;
}
.good-animation:hover {
  transform: scaleY(1.5);
}
```

### 4. 避免动态插入内容

```html
<!-- 坏：广告或弹窗突然出现，推动内容下移 -->
<div id="ad-slot"></div>
<script>
  // 页面加载后插入广告
  document.getElementById('ad-slot').innerHTML = '<iframe src="...">';
</script>

<!-- 好：预留空间 -->
<div id="ad-slot" style="min-height: 250px;">
  <!-- 广告加载后填充 -->
</div>
```

## Lighthouse 审计

### 命令行运行

```bash
# 安装 Lighthouse CLI
npm install -g lighthouse

# 运行审计
lighthouse http://localhost:3000 --output html --output-path ./lighthouse-report.html

# 只跑性能审计
lighthouse http://localhost:3000 --only-categories=performance --view
```

### Chrome DevTools

1. 打开 DevTools → Lighthouse 面板
2. 选择 "Performance" 类别
3. 点击 "Analyze page load"
4. 查看 Opportunities 和 Diagnostics 建议

### 关键审计项

| 审计项 | 说明 | 优化方法 |
| --- | --- | --- |
| LCP | 首屏最大内容 | 预加载 LCP 资源、优化图片 |
| Eliminate render-blocking resources | 渲染阻塞资源 | 内联关键 CSS、defer JS |
| Minify CSS/JS | 代码压缩 | 生产构建自动处理 |
| Serve images in next-gen formats | 现代图片格式 | WebP/AVIF |
| Efficiently encode images | 图片压缩 | 压缩工具 |
| Preconnect to required origins | 预连接 | `<link rel="preconnect">` |
| Reduce JS execution time | JS 执行时间 | 代码分割、减少 polyfill |
| Avoid multiple page redirects | 重定向 | 避免不必要的重定向 |
| Use HTTP/2 | 多路复用 | 服务器配置 H2/H3 |
| Passes `preload` audit | 预加载关键资源 | `<link rel="preload">` |

## 小结

性能优化的优先级：

1. **测量** — 先用 Lighthouse 和 Web Vitals 找到瓶颈
2. **LCP** — 预加载首屏关键资源、优化图片和字体
3. **CLS** — 为所有媒体预留空间、使用 transform 动画
4. **INP** — 分解长任务、防抖节流、React useTransition
5. **代码分割** — 按路由和交互懒加载
6. **持续监控** — 采集真实用户的 Web Vitals 数据

性能优化不是追求 100 分，而是在用户体验和开发成本之间找到平衡点。先解决 Poor 的问题，再逐步向 Good 靠拢。
